# 20. 【進階】CDN 模式下的組件抽離與豐富實作

在 ASP.NET Core MVC 或傳統 MPA 專案中，使用 Vue 3 CDN 時最常遇到的挑戰是「如何保持代碼整潔」以及「如何跨頁面複用組件」。本篇教學將介紹如何完美抽離組件，並實作一個具備 Props、Emits 與 Slots 的豐富組件。

---

## 核心策略：外部 JS 檔案 + 樣板字串 (Template Strings)

在沒有 Vite/Webpack 的環境下，我們無法使用 `.vue` 單檔案組件 (SFC)。最佳實踐是將組件定義在獨立的 `.js` 檔案中，並利用 JavaScript 的「樣板字串」來撰寫 HTML。

### 步驟 1：建立外部組件檔案 `AppCard.js`

建議存放路徑：`wwwroot/js/components/AppCard.js`

```javascript
/**
 * 豐富的卡片組件 (Rich Card Component)
 * 功能：支援自定義標題、類型、具名插槽、自定義事件
 */
const AppCard = {
  // 1. 屬性定義 (Props)
  props: {
    title: { 
        type: String, 
        default: '預設標題' 
    },
    type: { 
        type: String, 
        default: 'primary' // primary, success, danger, warning, info
    },
    showFooter: { 
        type: Boolean, 
        default: true 
    }
  },

  // 2. 自定義事件宣告 (Emits) - 提升可讀性與維護性
  emits: ['action-click'],

  // 3. 組件樣板 (Template) - 使用反引號支援多行 HTML
  template: `
    <div class="card mb-4 shadow-sm" :class="'border-' + type">
      <!-- 標題區域 -->
      <div class="card-header d-flex justify-content-between align-items-center" 
           :class="'bg-' + type + ' text-white'">
        <h5 class="mb-0">
            <i class="bi bi-info-circle me-2"></i>{{ title }}
        </h5>
        <!-- 具名插槽：讓父層可以在標題右側插入按鈕或標籤 -->
        <slot name="header-action"></slot>
      </div>
      
      <!-- 內容區域 -->
      <div class="card-body">
        <!-- 預設插槽：放置主要的 HTML 內容 -->
        <slot>
            <p class="text-muted">（目前沒有內容）</p>
        </slot>
      </div>
      
      <!-- 底部按鈕區域 -->
      <div v-if="showFooter" class="card-footer bg-transparent border-top-0 text-end">
        <hr class="mt-0">
        <button class="btn btn-sm" 
                :class="'btn-' + type" 
                @click="onButtonClick">
          詳細資訊 <i class="bi bi-arrow-right-short"></i>
        </button>
      </div>
    </div>
  `,

  // 4. 組件邏輯 (Methods)
  methods: {
    onButtonClick() {
      // 邏輯處理後，向父組件發送事件並攜帶資料
      this.$emit('action-click', { 
          title: this.title, 
          timestamp: new Date().toISOString() 
      });
    }
  }
};

// 將組件掛載到 window，方便全域引用
window.AppCard = AppCard;
```

### 步驟 2：在 .cshtml / .html 中引用與註冊

在您的頁面中，按照以下順序引入腳本：

```html
<!-- 1. 引入必要的 CSS (如 Bootstrap) -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/css/bootstrap.min.css" rel="stylesheet">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap-icons@1.8.1/font/bootstrap-icons.css">

<div id="app" class="container py-5">
    <h2 class="mb-4">Vue 3 組件抽離實戰</h2>

    <div class="row">
        <!-- 範例 A：基本使用 -->
        <div class="col-md-6">
            <app-card 
                title="系統監控" 
                type="success" 
                @action-click="handleEvent">
                
                <!-- 預設插槽內容 -->
                <div class="d-flex align-items-center">
                    <h1 class="display-4 text-success me-3">98%</h1>
                    <p>伺服器運行狀態良好，目前負載極低。</p>
                </div>
            </app-card>
        </div>

        <!-- 範例 B：使用具名插槽與進階屬性 -->
        <div class="col-md-6">
            <app-card 
                title="安全性警報" 
                type="danger" 
                :show-footer="true"
                @action-click="handleEvent">
                
                <!-- 具名插槽：header-action -->
                <template #header-action>
                    <span class="badge rounded-pill bg-light text-danger">緊急</span>
                </template>

                <!-- 預設插槽內容 -->
                <p>偵測到未經授權的存取嘗試：</p>
                <ul class="list-group list-group-flush">
                    <li class="list-group-item">IP: 192.168.1.105</li>
                    <li class="list-group-item">來源: 未知地區</li>
                </ul>
            </app-card>
        </div>
    </div>
</div>

<!-- 2. 引入 Vue 3 CDN -->
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<!-- 3. 引入抽離的組件檔案 -->
<script src="/js/components/AppCard.js"></script>

<!-- 4. 初始化 Vue App -->
<script>
    const { createApp } = Vue;

    const app = createApp({
        data() {
            return {
                eventLog: []
            }
        },
        methods: {
            handleEvent(data) {
                console.log('父組件收到資料:', data);
                alert(`觸發組件：${data.title}\n時間：${data.timestamp}`);
            }
        }
    });

    // 【關鍵】向 Vue App 註冊全域組件
    app.component('app-card', AppCard);

    app.mount('#app');
</script>
```

## 為什麼這樣做是「完美」的抽離？

1.  **複用性 (Reusability)**：`AppCard.js` 可以被專案中任何頁面引用，不需要在每個頁面重複寫 template。
2.  **維護性 (Maintainability)**：修改卡片的樣式或行為時，只需要修改一個檔案，全站同步更新。
3.  **零建置 (Zero Build)**：完全不需要安裝 Node.js、Vite 或 Webpack，直接在瀏覽器運行。
4.  **清晰的通訊結構**：
    *   **Props Down**：父層透過屬性決定子組件的顯示（顏色、標題）。
    *   **Events Up**：子組件點擊後通知父層處理邏輯（如跳轉、存檔）。
    *   **Slots Content**：父層靈活決定卡片中間要放什麼內容（文字、清單、圖表）。

## 常見問題 (Tips)

*   **IDE 語法高亮**：在 `.js` 檔案中寫 HTML 樣板時，可以使用 VS Code 的外掛如 `es6-string-html` 來獲得代碼高亮。
*   **路徑問題**：在 ASP.NET Core 中，請確保路徑指向 `wwwroot` 下的正確位置。
*   **多個組件**：若有多個組件，可以建立一個 `components.js` 統一打包，或按功能分開存放。

## 進階：組件連動與 $refs 操作

在複雜的介面中，我們常需要「組件 A 影響組件 B」（連動），或「父層直接呼叫子組件的方法」（$refs）。

### 1. 使用 $refs 呼叫子組件方法 (以 Modal 為例)

當我們需要操作像「彈窗 (Modal)」這種具有內部狀態（顯示/隱藏）的組件時，`$refs` 是最直接的工具。

#### 建立 `AppModal.js`
```javascript
const AppModal = {
  props: ['title'],
  template: `
    <div class="modal fade" ref="bootstrapModal" tabindex="-1">
      <div class="modal-dialog">
        <div class="modal-content">
          <div class="modal-header">
            <h5 class="modal-title">{{ title }}</h5>
            <button type="button" class="btn-close" @click="hide"></button>
          </div>
          <div class="modal-body">
            <slot></slot>
          </div>
          <div class="modal-footer">
            <button type="button" class="btn btn-secondary" @click="hide">關閉</button>
            <slot name="footer-btns"></slot>
          </div>
        </div>
      </div>
    </div>
  `,
  data() {
    return { modalInstance: null };
  },
  mounted() {
    // 初始化 Bootstrap Modal 實例 (需引入 Bootstrap JS)
    this.modalInstance = new bootstrap.Modal(this.$refs.bootstrapModal);
  },
  methods: {
    show() { this.modalInstance.show(); },
    hide() { this.modalInstance.hide(); }
  }
};
window.AppModal = AppModal;
```

### 2. 組件連動 (以搜尋同步為例)

組件連動的核心原則是：**狀態提升 (Lifting State Up)**。將資料存在父層，透過 Props 傳給子組件 A，並透過 Events 接收子組件 B 的變更。

#### 建立 `AppSearch.js`
```javascript
const AppSearch = {
  props: ['modelValue'], // 支援 v-model
  emits: ['update:modelValue'],
  template: `
    <div class="input-group mb-3">
      <span class="input-group-text"><i class="bi bi-search"></i></span>
      <input type="text" class="form-control" 
             placeholder="輸入關鍵字搜尋..."
             :value="modelValue"
             @input="$emit('update:modelValue', $event.target.value)">
    </div>
  `
};
window.AppSearch = AppSearch;
```

### 3. 完整整合範例

在父頁面中展示如何連動這兩個組件：

```html
<div id="app" class="container py-5">
    <!-- A 組件：搜尋列 (與父層 searchQuery 連動) -->
    <app-search v-model="searchQuery"></app-search>

    <div class="card p-3 mb-3 border-start border-5 border-info">
        <p class="mb-2">目前搜尋關鍵字：<strong class="text-primary">{{ searchQuery || '（尚未輸入）' }}</strong></p>
        
        <!-- 點擊後透過 $refs 觸發 B 組件 -->
        <button class="btn btn-dark" @click="openConfirmModal" :disabled="!searchQuery">
            確認搜尋內容
        </button>
    </div>

    <!-- B 組件：彈窗 (由父層透過 ref 控制) -->
    <app-modal ref="searchModal" title="搜尋確認">
        您確定要以「<strong>{{ searchQuery }}</strong>」作為關鍵字進行系統檢索嗎？
        <template #footer-btns>
            <button class="btn btn-primary" @click="confirmSearch">開始搜尋</button>
        </template>
    </app-modal>
</div>

<!-- 引入 Bootstrap JS (Modal 必備) -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.1.3/dist/js/bootstrap.bundle.min.js"></script>

<script src="/js/components/AppSearch.js"></script>
<script src="/js/components/AppModal.js"></script>

<script>
    const { createApp } = Vue;
    createApp({
        data() {
            return {
                searchQuery: ''
            }
        },
        methods: {
            openConfirmModal() {
                // 使用 $refs 直接呼叫子組件定義的 show 方法
                this.$refs.searchModal.show();
            },
            confirmSearch() {
                alert('系統正在檢索: ' + this.searchQuery);
                this.$refs.searchModal.hide();
            }
        }
    })
    .component('app-search', AppSearch)
    .component('app-modal', AppModal)
    .mount('#app');
</script>
```

## 總結進階技巧

1.  **$refs 的使用時機**：僅用於呼叫子組件的「動作」（如 `show()`, `focus()`, `reset()`）。不要用它來傳遞大量資料。
2.  **v-model 組件化**：透過 `modelValue` 與 `update:modelValue`，自定義組件能完美融入 Vue 的雙向綁定生態。
3.  **封裝第三方套件**：在 `mounted` 中初始化如 Modal、Swiper、Chart.js 等非 Vue 套件，是將舊有專案升級至 Vue 的最佳過渡方案。

## 補充：大師級組件開發指南

為了讓組件在大型專案中更穩健，您還需要掌握以下三個知識點：

### 1. 樣板太長？使用 X-Template 分離
當組件樣板超過 50 行時，寫在 JS 的反引號內會變得很難看。您可以將 HTML 留在 `.cshtml` 中，僅將邏輯抽離。

**在 HTML 中：**
```html
<script type="text/x-template" id="tpl-complex-table">
  <div class="table-responsive">
    <table class="table table-hover">
       <!-- 這裡可以寫極其複雜的 HTML，享有 IDE 的語法高亮 -->
    </table>
  </div>
</script>
```

**在 JS 中：**
```javascript
const ComplexTable = {
  template: '#tpl-complex-table', // 直接指向 ID
  props: ['data'],
  // ... 其他邏輯
};
```

### 2. 解決樣式衝突：CSS 命名空間
由於沒有 `.vue` 檔案的 `<style scoped>`，組件樣式建議採用 **Prefix (前綴)** 命名法。

```css
/* 建議做法：為組件建立專屬前綴 */
.v-app-card { ... }
.v-app-card .card-body { ... }
.v-app-card-header { ... }
```
並在組件 template 的最外層加上該 Class：
```javascript
template: `<div class="card v-app-card"> ... </div>`
```

### 3. 組件自給自足：生命週期與 API
優質的組件應該具備「自動載入資料」的能力，而不是所有資料都由父層傳入。

```javascript
const UserProfile = {
  props: ['userId'],
  template: `
    <div v-if="loading">載入中...</div>
    <div v-else class="user-info">
      <h4>{{ user.name }}</h4>
    </div>
  `,
  data() {
    return { user: null, loading: true };
  },
  async mounted() {
    // 組件掛載後自動抓取資料
    try {
      const response = await fetch(`/api/user/${this.userId}`);
      this.user = await response.json();
    } finally {
      this.loading = false;
    }
  }
};
```

### 4. 邏輯重用：提取純 JS 函數
如果您有多個組件需要相同的計算邏輯（例如：日期格式化、權限檢查），請將其提取為普通的 JS 模組，而不是寫死在組件內。

```javascript
// helpers.js
const DateHelper = {
  format(date) { return new Date(date).toLocaleDateString(); }
};

// Component.js
const MyComponent = {
  template: `<div>{{ formatDate(item.date) }}</div>`,
  methods: {
    formatDate(d) {
      return DateHelper.format(d); // 調用外部邏輯
    }
  }
};
```

## 最終檢查清單 (Final Checklist)

*   [ ] **組件是否獨立？** 修改組件內部不應該導致父層崩潰。
*   [ ] **命名是否衝突？** 是否使用了全域唯一的組件名稱（如 `app-xxx`）。
*   [ ] **Props 是否有預設值？** 確保沒有傳入參數時，組件依然能正常顯示。
*   [ ] **事件命名是否規範？** 建議使用 `update:xxx` 或 `on-xxx` 保持一致性。
