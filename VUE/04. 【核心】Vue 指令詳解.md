# **重構版：4. 【核心】Vue 指令詳解**

指令 (Directives) 是 Vue 模板中帶有 `v-` 前綴的特殊屬性。它們是 Vue 響應式系統與真實 DOM 之間的橋樑，讓我們可以根據資料的狀態，動態地操作 DOM 的結構、樣式或行為。

本章將全面梳理 Vue 3 的內建指令，並提供豐富的範例，協助您在 `.cshtml` 開發中運用自如。

---

## **4.1 指令總覽**

Vue 3 提供的核心指令可以分為以下幾類：

1.  **條件渲染**：`v-if`, `v-else`, `v-else-if`, `v-show`
2.  **列表渲染**：`v-for`
3.  **屬性與事件綁定**：`v-bind` (:), `v-on` (@)
4.  **表單雙向綁定**：`v-model`
5.  **內容渲染**：`v-text`, `v-html`
6.  **效能與特殊優化**：`v-once`, `v-memo`, `v-pre`, `v-cloak`, `v-slot`

---

## **4.2 條件渲染：控制元素的「存在」與「顯示」**

### **4.2.1 `v-if`, `v-else-if`, `v-else`**
這是「真正」的條件渲染，如果條件不成立，該元素及其子元素**完全不會被渲染**在 DOM 中。

```html
<div id="app">
    <div v-if="userRole === 'admin'">
        <p>歡迎管理員！您可以存取所有設定。</p>
    </div>
    <div v-else-if="userRole === 'editor'">
        <p>您好編輯，您可以撰寫與發佈文章。</p>
    </div>
    <div v-else>
        <p>一般訪客，歡迎瀏覽內容。</p>
    </div>
</div>
```

### **4.2.2 `v-show`**
元素**永遠會被渲染**並保留在 DOM 中，只是簡單地切換 CSS 的 `display` 屬性。

*   **何時使用？** 如果元素需要非常頻繁地顯示/隱藏，使用 `v-show` 效能更好。
*   **注意**：`v-show` 不支援 `<template>` 元素，也不支援 `v-else`。

```html
<!-- 適用於頻繁切換的選單或彈窗 -->
<div v-show="isMenuOpen" class="dropdown-menu">
    這是下拉選單內容
</div>
```

## **4.3 列表渲染：`v-for`**

用於遍歷陣列、物件或數字來渲染一組元素。

**關鍵點：`:key` 是必須的**。它幫助 Vue 識別每個節點的身份，從而高效地更新虛擬 DOM。

```html
<ul>
    <!-- 1. 遍歷陣列 -->
    <li v-for="(item, index) in items" :key="item.id">
        {{ index }} - {{ item.name }}
    </li>

    <!-- 2. 遍歷物件 -->
    <li v-for="(value, key, index) in userProfile" :key="key">
        {{ index }}. {{ key }}: {{ value }}
    </li>

    <!-- 3. 指定次數的循環 (從 1 到 10) -->
    <li v-for="n in 10" :key="n">{{ n }}</li>
</ul>
```

> **重要陷阱：v-if 與 v-for 同時使用**
> 在 Vue 3 中，`v-if` 的優先級高於 `v-for`。這意味著 `v-if` 無法存取 `v-for` 範圍內的變數。
> **正確做法**：使用計算屬性 (computed) 先過濾列表，或使用 `<template>` 標籤包裹 `v-for`。

## **4.4 屬性與事件綁定：`v-bind` 與 `v-on`**

這兩個指令在第 3 章已有介紹，這裡補充**動態參數**與**物件綁定**。

### **4.4.1 `v-bind` (簡寫 `:`)**
```html
<!-- 1. 基本屬性綁定 -->
<img :src="imageSrc">

<!-- 2. 動態屬性名稱 (Vue 2.6+) -->
<!-- 如果 attributeName 是 "href"，則等同於 :href="url" -->
<a :[attributeName]="url"> 連結 </a>

<!-- 3. 綁定一個物件 (一次設定多個屬性) -->
<div v-bind="objectOfAttrs"></div>
```
```javascript
data() {
    return {
        attributeName: 'title',
        url: 'https://vuejs.org',
        objectOfAttrs: {
            id: 'container',
            class: 'wrapper',
            'data-id': 123
        }
    }
}
```

### **4.4.2 `v-on` (簡寫 `@`)**
```html
<!-- 1. 動態事件名稱 -->
<button @[eventName]="doSomething"></button>

<!-- 2. 按鍵修飾符：只有在按下 Enter 時才觸發 -->
<input @keyup.enter="submitForm">

<!-- 3. 系統修飾鍵：Ctrl + Click -->
<div @click.ctrl="doSomething">Ctrl + Click Me</div>
```

## **4.5 表單雙向綁定：`v-model`**

`v-model` 是語法糖，會根據表單元素自動監聽使用者的輸入並更新資料。

### **4.5.1 各種表單元素**
```html
<!-- 文字輸入 -->
<input v-model="text">

<!-- 多行文字 -->
<textarea v-model="message"></textarea>

<!-- 單選按鈕 -->
<input type="radio" value="A" v-model="picked"> A
<input type="radio" value="B" v-model="picked"> B

<!-- 複選框 (綁定到陣列) -->
<input type="checkbox" value="Jack" v-model="checkedNames">
<input type="checkbox" value="John" v-model="checkedNames">

<!-- 下拉選單 -->
<select v-model="selected">
    <option disabled value="">請選擇</option>
    <option>台北</option>
    <option>台中</option>
</select>
```

### **4.5.2 指令修飾符 (必學！)**
*   **`.lazy`**：將監聽事件從 `input` 改為 `change`（失去焦點後才更新）。
*   **`.number`**：自動將使用者輸入值轉為數值型態。
*   **`.trim`**：自動過濾使用者輸入的首尾空白字元。

```html
<input v-model.number.trim="age" type="number">
<p>年齡 (型別為 number): {{ age }}</p>
```

## **4.6 內容渲染與優化**

### **4.6.1 `v-text` 與 `v-html`**
*   `v-text`：更新元素的 `textContent` (純文字)。
*   `v-html`：更新元素的 `innerHTML` (**注意 XSS 風險**)。

### **4.6.2 `v-once`**
只渲染一次。隨後的更新將跳過該元素及其子元素，用於優化不變的靜態內容。
```html
<span v-once>這條訊息永遠不會變: {{ message }}</span>
```

### **4.6.3 `v-memo` (Vue 3.2+ 新增)**
這是一個高效能優化指令。它接受一個依賴陣列，只有當陣列中的值發生變化時，才會重新渲染該元素及其子樹。
**場景**：在非常巨大的 `v-for` 列表中，如果只需要在某些條件下更新列表項。

```html
<!-- 只有當 item.id 或 item.isUpdated 變化時才重新渲染這行 -->
<div v-for="item in list" :key="item.id" v-memo="[item.id, item.isUpdated]">
    {{ item.name }} - {{ item.status }}
</div>
```

### **4.6.4 `v-pre`**
跳過該元素及其子元素的編譯。直接顯示原始標籤，用於顯示 Vue 語法本身。
```html
<span v-pre>{{ 這不會被編譯 }}</span>
```

### **4.6.5 `v-cloak`**
保持在元素上直到關聯實例結束編譯。配合 CSS `[v-cloak] { display: none; }` 解決 CDN 模式下的標籤閃爍問題。

## **4.7 實戰範例：綜合應用**

這是一個結合了多個指令的搜尋與過濾範例：

```html
<div id="app" v-cloak>
    <h3>人員名單管理</h3>
    
    <!-- v-model.trim 雙向綁定搜尋詞 -->
    <input v-model.trim="searchQuery" placeholder="搜尋姓名...">

    <!-- v-if 條件顯示 -->
    <p v-if="filteredList.length === 0">找不到相符的人員。</p>

    <ul v-else>
        <!-- v-for 列表渲染 -->
        <!-- v-bind:class 動態樣式 -->
        <li v-for="person in filteredList" 
            :key="person.id"
            :class="{ 'highlight': person.age >= 18 }">
            
            {{ person.name }} ({{ person.age }} 歲)
            
            <!-- v-on 事件處理 -->
            <button @click="removePerson(person.id)">刪除</button>
        </li>
    </ul>
</div>
```

```javascript
const app = Vue.createApp({
    data() {
        return {
            searchQuery: '',
            persons: [
                { id: 1, name: '小明', age: 15 },
                { id: 2, name: '老王', age: 40 },
                { id: 3, name: '張三', age: 22 }
            ]
        }
    },
    computed: {
        filteredList() {
            return this.persons.filter(p => 
                p.name.includes(this.searchQuery)
            );
        }
    },
    methods: {
        removePerson(id) {
            this.persons = this.persons.filter(p => p.id !== id);
        }
    }
});
app.mount('#app');
```

## **總結**

| 指令 | 縮寫 | 用途 | 重點 |
| :--- | :--- | :--- | :--- |
| `v-bind` | `:` | 綁定 HTML 屬性 | 支援動態參數與物件綁定 |
| `v-on` | `@` | 監聽事件 | 修飾符 (.prevent, .stop) |
| `v-model` | | 雙向資料綁定 | 修飾符 (.lazy, .number, .trim) |
| `v-for` | | 循環渲染 | 必須搭配 `:key` |
| `v-if` | | 條件渲染 (DOM 移除) | 優先級高於 `v-for` |
| `v-show` | | 顯示/隱藏 (CSS 切換) | 頻繁切換時使用 |
| `v-text` / `v-html` | | 渲染純文字或 HTML | `v-html` 需注意 XSS 風險 |
| `v-slot` | `#` | 定義插槽內容 | 詳細用法請參考第 9 章 |
| `v-memo` | | 效能快取 (3.2+) | 大數據列表渲染優化 |
| `v-cloak` | | 隱藏未編譯模板 | CDN 開發必備 |
| `v-once` / `v-pre` | | 效能優化 / 跳過編譯 | 處理靜態內容與原始語法 |
