# 04. 看板方法與視覺化 (Kanban)

看板 (Kanban) 的核心意義在於「視覺化」與「限制在製品 (WIP)」。讓所有組員一眼就能看出專題現在的進度，以及誰卡住了。

## 1. 基礎看板結構

一個最簡單的看板包含三個欄位：

| To Do (待辦) | Doing (進行中) | Done (已完成) |
| :--- | :--- | :--- |
| (還沒開始的故事) | (正在寫程式的故事) | (通過測試驗收的故事) |

## 2. 數位工具選擇

學生專題建議使用免費且協作方便的工具：

*   **Trello**: 最直覺，卡片式拖拉。
*   **GitHub Projects**: 對於程式開發最方便，可以直接與 Issue 和 Pull Request 連動。
*   **Notion**: 如果你們已經用 Notion 做筆記，可以直接開 Board View。

## 3. 看板運作規則 (Rules)

1.  **視覺化**: 所有的工作 (Coding, 設計, 寫文件) 都要變成卡片放上去。
2.  **限制 WIP (Work In Progress)**:
    *   不要同時開太多戰場！
    *   **原則**: "Stop Starting, Start Finishing." (停止開始新工作，專注完成手邊工作)。
    *   例如：規定 Doing 欄位同一時間只能有 3 張卡片。如果滿了，大家要先去幫忙把卡住的工作做完，才能拉新的。
3.  **定義 "Done" (DoD - Definition of Done)**:
    *   怎樣才算移到 Done？
    *   例如：程式碼寫完 + 通過編譯 + 上傳到 GitHub + 在測試機上能跑。

---

## 4. 專題實作練習 (Homework)

### 任務：建立專題看板
1.  **開立看板**: 註冊 Trello 或使用 GitHub Projects。
2.  **設定欄位**: 建議欄位：`Backlog` -> `To Do (Sprint)` -> `Doing` -> `Testing/Review` -> `Done`。
3.  **填入工作**: 將剛剛 Sprint Planning 選出的 User Stories 建立成卡片，放在 `To Do`。
4.  **分工**: 每個人認領一張卡片，將頭像拉上去，並將卡片移到 `Doing`。

**注意**: 每日站立會議時，請看著這個看板開會！
